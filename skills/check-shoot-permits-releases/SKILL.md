# Check Shoot Permits and Releases

Provide issue spotting and workflow support, not legal conclusions. Recommend qualified local counsel or the relevant authority for uncertain or high-value use.

## Workflow

1. Identify jurisdiction, public/private property, commercial/editorial purpose, crew footprint, equipment, drone, vehicles, structures, amplified sound, generators, animals, pyrotechnics and street/sidewalk impact.
2. Identify recognizable people, minors, private property, artwork, trademarks, products, uniforms and confidential information.
3. Record intended media, territory, term, exclusivity, paid advertising, sublicensing, AI/model-training use, editing/derivative rights and compensation.
4. Call `generate_release_checklist` to produce a case-specific matrix.
5. Track document owner, counterparty, sent/signed dates, restrictions, expiration and file location.
6. Verify certificate-of-insurance limits/additional insured language and permit conditions with the recipient.

## Status

Use `not assessed`, `likely required—verify`, `requested`, `received`, `signed`, `approved with restrictions`, `expired`, or `not required—authority confirmed`.

Never state that a release or permit is unnecessary based only on general web guidance. Never draft deceptive or coercive consent language.
