# Create Photo Shot List

Map each approved deliverable to observable frames. Keep creative flexibility inside explicit coverage requirements.

## Fields

Assign each shot: unique ID; deliverable/use; priority (`must`, `should`, `nice`); setup; subject/talent; action/expression; wardrobe/HMU; prop/product; location/background; framing; orientation/aspect ratio; lens/perspective; lighting; notes; estimated minutes; approval owner; completion status.

## Workflow

1. Extract required assets from the brief and channel specifications.
2. Decompose hero concepts into safe coverage, detail, alternate expression/action, and crop variants.
3. Group by setup to reduce lighting, wardrobe, set and location changes.
4. Put must-haves early enough to survive delays.
5. Estimate setup change separately from capture time.
6. Add product, label, continuity, legal-copy and negative-space checks where applicable.
7. Reconcile totals against the call sheet; cut or downgrade shots when the schedule cannot fit.

## Audit

Flag unmapped deliverables, duplicate shots, missing orientations, impossible dependencies, undefined approvals, excessive setup churn and schedules with no buffer.
